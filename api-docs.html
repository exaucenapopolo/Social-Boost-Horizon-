Introduction

L'API Social Boost Horizon vous permet de passer des commandes de services de boost de reseaux sociaux directement depuis votre application.

Devise

Tous les prix et montants sont exprimes en XAF (FCFA).

URL de base

Toutes les requetes API doivent etre envoyees a l'URL suivante :

https://social-boost-exaucenapopolo2.replit.app

Authentification

Chaque requete doit inclure votre cle API dans le header HTTP X-Api-Key.

Vous pouvez generer votre cle API depuis votre page de profil.

Exemple de header

X-Api-Key: votre_cle_api_ici

Verifier le solde

GET /api/v1/balance

Recupere le solde actuel de votre compte.

Reponse

200 OK
{
  "success": true,
  "balance": 15000,
  "currency": "XAF"
}
curl -X GET https://social-boost-exaucenapopolo2.replit.app/api/v1/balance \
  -H "X-Api-Key: votre_cle_api"
import requests

url = "https://social-boost-exaucenapopolo2.replit.app/api/v1/balance"
headers = {"X-Api-Key": "votre_cle_api"}

response = requests.get(url, headers=headers)
print(response.json())
$ch = curl_init();

curl_setopt_array($ch, [
    CURLOPT_URL => "https://social-boost-exaucenapopolo2.replit.app/api/v1/balance",
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_HTTPHEADER => [
        "X-Api-Key: votre_cle_api"
    ]
]);

$response = curl_exec($ch);
curl_close($ch);

echo $response;
const response = await fetch("https://social-boost-exaucenapopolo2.replit.app/api/v1/balance", {
    headers: {
        "X-Api-Key": "votre_cle_api"
    }
});

const data = await response.json();
console.log(data);

Passer une commande

POST /api/v1/order

Passe une nouvelle commande de service de boost.

Corps de la requete (JSON)

ParametreTypeDescription
service_id requis integer ID du service (obtenu via /api/v1/services)
link requis string Lien du profil ou de la publication cible
quantity requis integer Quantite souhaitee (entre min et max du service)
comments optionnel string Commentaires personnalises (si le service le supporte)
provider optionnel string Fournisseur : exo, mtp ou smmgen (defaut: exo)

Reponse

200 OK
{
  "success": true,
  "order_id": "ORD-1234567890",
  "provider_order_id": 98765,
  "provider": "exo",
  "service": "Instagram Followers",
  "quantity": 1000,
  "price": 5000,
  "currency": "XAF",
  "balance": 10000,
  "status": "pending"
}
curl -X POST https://social-boost-exaucenapopolo2.replit.app/api/v1/order \
  -H "X-Api-Key: votre_cle_api" \
  -H "Content-Type: application/json" \
  -d '{
    "service_id": 1,
    "link": "https://instagram.com/username",
    "quantity": 1000,
    "provider": "exo"
  }'
import requests

url = "https://social-boost-exaucenapopolo2.replit.app/api/v1/order"
headers = {
    "X-Api-Key": "votre_cle_api",
    "Content-Type": "application/json"
}
data = {
    "service_id": 1,
    "link": "https://instagram.com/username",
    "quantity": 1000,
    "provider": "exo"
}

response = requests.post(url, headers=headers, json=data)
print(response.json())
$ch = curl_init();

$payload = json_encode([
    "service_id" => 1,
    "link" => "https://instagram.com/username",
    "quantity" => 1000,
    "provider" => "exo"
]);

curl_setopt_array($ch, [
    CURLOPT_URL => "https://social-boost-exaucenapopolo2.replit.app/api/v1/order",
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_POST => true,
    CURLOPT_POSTFIELDS => $payload,
    CURLOPT_HTTPHEADER => [
        "X-Api-Key: votre_cle_api",
        "Content-Type: application/json"
    ]
]);

$response = curl_exec($ch);
curl_close($ch);

$data = json_decode($response, true);
print_r($data);
const response = await fetch("https://social-boost-exaucenapopolo2.replit.app/api/v1/order", {
    method: "POST",
    headers: {
        "X-Api-Key": "votre_cle_api",
        "Content-Type": "application/json"
    },
    body: JSON.stringify({
        service_id: 1,
        link: "https://instagram.com/username",
        quantity: 1000,
        provider: "exo"
    })
});

const data = await response.json();
console.log(data);

Gestion des erreurs

L'API retourne des codes HTTP standards. En cas d'erreur, la reponse contient un objet JSON avec les details :

Exemple d'erreur
{
  "success": false,
  "error": "Cle API manquante ou invalide"
}

Codes d'erreur

CodeDescription
401 Cle API manquante ou invalide
400 Parametres manquants ou invalides, solde insuffisant
404 Commande non trouvee
502 Erreur fournisseur (la commande est remboursee automatiquement)
500 Erreur serveur
